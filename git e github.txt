Git - Hackerhank

O que é repositório de código fonte?

Sistema voltado para trabalho em equipe que visa resolver o problema de trabalho em arquivos compartilhados.

Podemos imaginar uma equipe de duas pessoas alterando o mesmo arquivo. O último que salvar, substituirá o primeiro que foi salvo, sem se quer avisar. Trabalhando com repositório, o sistema avisa quando houver esse tipo conflito além de oferecer recursos para que possamos tratar esses tipos de erros sem perder qualquer informação.

O que é o Git?

É um sistema de controle de versão. 
Com o git instalado o computador passa a funcionar como se fosse um servidor individual para gerenciamento dos arquivos, também, de forma individual.


Github

Funciona como outro servidor em domínio geral, onde funciona online.

Imagina que cada computador com o git instalado; um projeto onde três pessoas irão trabalhar nele; Agora, cada um terá seu papel atribuído dentro do projeto; Cada qual começa a desenvolver suas atividades de forma específica.

Com isso em mente, teremos que centralizar todas essas alterações forma a manter o projeto atualizado com todas as alterações em um só lugar.

É aí que entra o gerenciador externo de repositórios, o Github.


1. Criando um respositório de projeto no github
- github.com
- criar ou logar com uma conta
- criar um respositório (nome, descrição e inicar com readme)
- Copiar o HTTPS com o endereço do projeto. (para clonar)
- No nosso computador, com o git instalado, abrir uma pasta onde serão colocados todos os projetos
- Execute o cmd

2. Comandos:

- GIT CLONE
Na linha de comando, digite: git clone e cole o endereço do projeto do github:
	git clone https://github.com/emersonps/teste_comandos.git

no final, veremos uma pasta do projeto do github no nosso local.

3. Enviando arquivos e alterações para o github
- Cria um arquido de texto dentro da pasta do projeto que copiamos.
- Com isso, temos em nosso local, a cópia do projeto que está no gihub e agora uma alteração onde adicionamos um arquivo nesse projeto. Por hora, essa alteração está apenas dentro da cópia do projeto em nosso computador.
